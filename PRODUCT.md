# Product

## Register

product

## Users

Developers looking up GitHub profiles — their own or others'. Used at a desk, full attention, likely on a large monitor. The primary job: get a quick but complete picture of a GitHub user's activity and repo landscape without navigating GitHub's own UI.

## Product Purpose

A GitHub analytics dashboard. Enter a username, get a visual breakdown of repos, languages, contribution activity, and stats. Faster and more useful than GitHub's own profile view. Designed to be kept open and used regularly.

## Brand Personality

Sharp, warm, technical. Confident without being cold. Feels like a tool a senior engineer built for themselves and decided to share — not a startup product trying to impress investors.

## Anti-references

- **GitHub itself** — this is an elevated take on GitHub data, not a GitHub clone. Should not look like it belongs inside github.com.
- **Generic SaaS** — no purple gradient heroes, no hero-metric templates, no glassmorphism cards, no "AI made this" visual fingerprints.

## Design Principles

1. **Data is the UI** — every element on screen should be there because it helps the user understand something. Decoration that doesn't carry information gets cut.
2. **Earned density** — the dashboard can be information-dense, but density must be legible. No crowding just to look technical.
3. **Warm precision** — sharp edges, exact spacing, but not clinical. Subtle choices (weight, spacing, tone) that make the tool feel like it was made by someone who cares.
4. **Confidence without flash** — no animations to impress, no gradients to fill space. Every motion and color choice earns its place.
5. **Developer-grade** — the user is a developer. They notice inconsistency. They notice when something is slightly off. Design to that standard.

## Accessibility & Inclusion

WCAG AA minimum. Keyboard navigable throughout. Color is never the sole signal for state (errors, status). Dark mode is the default; light mode is a real alternative, not an afterthought.
